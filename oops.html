1. What is Object-Oriented Programming?
OOP is a programming paradigm based on the concept of objects, which can contain data and methods to operate on that data.
2. What are the main principles of OOP?
  • Encapsulation
  • Abstraction
  • Inheritance
  • Polymorphism
3. What is a class?
A class is a blueprint for creating objects, defining properties and methods that the objects will have.
4. What is an object?
An instance of a class that represents a specific entity with defined attributes and behaviors.
5. What is encapsulation?
Encapsulation is the concept of bundling data and methods that operate on that data into a single unit (class) and restricting access to some of the object's components.
6. What is abstraction?
Abstraction hides complex implementation details and shows only the essential features of an object, simplifying interactions.
7. What is inheritance?
Inheritance allows a class to inherit properties and behaviors (methods) from another class, promoting code reuse.
8. What is polymorphism?
Polymorphism allows objects of different classes to be treated as objects of a common superclass, enabling method overriding and overloading.
9. What is method overloading?
Method overloading occurs when two or more methods in the same class have the same name but different parameters.
10. What is method overriding?
Method overriding allows a subclass to provide a specific implementation of a method already defined in its superclass.
11. What is the difference between overloading and overriding?
Overloading is resolving method calls based on different parameters within the same class; overriding involves redefining a superclass method in a subclass.
12. What is a constructor?
A constructor is a special method used to initialize objects when they are created.
13. What is a destructor?
A destructor is a method called when an object is destroyed, typically used for cleanup.
14. What is the difference between a constructor and a method?
A constructor is used for object initialization and is called automatically; a method defines behavior and is called explicitly.
15. What is a static method?
A static method belongs to the class rather than an instance, and it can be called without creating an object of the class.
16. What is a class variable?
A class variable is shared among all instances of a class, whereas an instance variable is unique to each object.
17. What is an instance variable?
An instance variable is a variable defined inside a class and is unique to each object created from the class.
18. What is a superclass?
A superclass (parent class) is a class that is inherited from by another class (subclass).
19. What is a subclass?
A subclass (child class) inherits from a superclass and can extend or override its behavior.
20. What is the this keyword?
The this keyword refers to the current instance of a class, used to differentiate instance variables from parameters.
21. What is the super keyword?
The super keyword refers to the superclass, often used to call a superclass's methods or constructor from a subclass.
22. What is an interface?
An interface defines a contract of methods that a class must implement, but it cannot provide method implementations.
23. What is an abstract class?
An abstract class cannot be instantiated and may contain abstract methods (methods without implementations) that must be implemented by subclasses.
24. What is the difference between an abstract class and an interface?
An abstract class can have implemented methods and is used for common behavior; an interface defines a contract with no implementation and is used for multiple inheritance.
25. What is multiple inheritance?
Multiple inheritance occurs when a class inherits from more than one class. Some languages (e.g., Java) do not support multiple inheritance to avoid complexity.
26. What is the difference between dynamic binding and static binding?
Dynamic binding occurs at runtime, typically for method overriding. Static binding occurs at compile-time, such as when calling methods of final or private classes.
27. What is an abstract method?
An abstract method is a method declared in an abstract class or interface, which must be implemented by subclasses.
28. What is the difference between an interface and an abstract class?
An interface only defines method signatures with no implementation, whereas an abstract class can have both abstract and concrete methods.
29. What is a package?
A package is a namespace that organizes a set of related classes and interfaces, making it easier to manage and reuse code.
30. What is the access modifier public?
A public modifier allows access to the class, method, or variable from any other class.
31. What is the access modifier private?
A private modifier restricts access to the class, method, or variable to only within the same class.
32. What is the access modifier protected?
A protected modifier allows access to the class, method, or variable from the same class, subclasses, and classes in the same package.
33. What is the final keyword?
The final keyword can be applied to variables, methods, and classes, making them immutable or preventing method overriding or inheritance.
34. What is the static keyword?
The static keyword allows class-level variables and methods that are shared across all instances of the class.
35. What is the transient keyword?
The transient keyword is used in Java to indicate that a variable should not be serialized.
36. What is the volatile keyword?
The volatile keyword ensures that a variable's value is always read from and written to the main memory, useful in multi-threaded programming.
37. What is the difference between shallow copy and deep copy?
A shallow copy duplicates the reference to objects, whereas a deep copy duplicates the object itself, including objects referenced by it.
38. What is a singleton class?
A singleton class is designed to allow only one instance of the class to exist at any time.
39. What is composition?
Composition is a design principle where one class contains instances of other classes to build complex types.
40. What is aggregation?
Aggregation is a type of association where one class can contain references to other classes, but the contained objects can exist independently.
41. What is the difference between composition and aggregation?
Composition implies strong ownership, where the contained object's lifecycle depends on the container. Aggregation implies weak ownership, where the contained object's lifecycle is independent.
42. What is method hiding?
Method hiding occurs when a subclass defines a static method with the same name as a static method in its superclass.
43. What is a nested class?
A nested class is a class defined within another class. It can be static or non-static, depending on its relation to the outer class.
44. What is a lambda expression in OOP?
Lambda expressions allow the creation of anonymous functions to provide a clear and concise way to represent one method interface using an expression.
45. What is a functional interface?
A functional interface has exactly one abstract method, and it can have multiple default or static methods. It is used primarily for lambda expressions.
46. What is the difference between a constructor and a method?
A constructor initializes an object when it is created and has the same name as the class, whereas a method performs operations on an object and is called explicitly.
47. What is the difference between static and instance methods?
Static methods belong to the class and can be called without creating an instance, while instance methods are tied to object instances and require an object to be invoked.
48. What is a default constructor?
A default constructor is a no-argument constructor automatically provided by the compiler if no other constructors are explicitly defined in a class.
49. What is a constructor overloading?
Constructor overloading occurs when a class has more than one constructor with different parameters to initialize objects in different ways.
50. What is a method reference?
A method reference is a shorthand notation for calling a method in Java using the :: operator, typically used with lambda expressions.
