Java Interview Questions

1. What is Java?

Java is a high-level, object-oriented programming language.

It is platform-independent due to the Java Virtual Machine (JVM).

2. What is JVM?

JVM (Java Virtual Machine) executes Java bytecode.

It provides platform independence and manages memory and security.

3. What is the difference between JDK and JRE?

JDK (Java Development Kit) includes tools to develop Java apps; JRE runs them.

JDK = JRE + development tools like javac.

4. What are Java's main features?

Java is platform-independent, object-oriented, secure, and robust.

It supports multithreading, garbage collection, and high performance.

5. What is bytecode?

Bytecode is an intermediate code generated by the Java compiler.

It is executed by the JVM on any platform.

6. What is a class in Java?

A class is a blueprint for creating objects.

It defines fields and methods that objects will have.

7. What is an object in Java?

An object is an instance of a class.

It holds actual values in memory for fields defined in the class.

8. What is inheritance?

Inheritance lets a class acquire properties of another class.

Java supports single, multilevel, and hierarchical inheritance.

9. What is polymorphism?

Polymorphism means taking many forms.

It is achieved via method overloading and method overriding.

10. What is encapsulation?

Encapsulation means wrapping data and methods into one unit.

It hides internal details using private fields and public methods.

11. What is abstraction?

Abstraction hides complex implementation details and shows only functionality.

It is achieved using abstract classes and interfaces.

12. What is method overloading?

Method overloading means defining multiple methods with the same name but different parameters.

It happens at compile time.

13. What is method overriding?

Method overriding means redefining a superclass method in a subclass.

It allows runtime polymorphism.

14. What is the difference between == and .equals()?

== checks reference equality; .equals() checks object content.

Use .equals() to compare values.

15. What is a constructor?

A constructor is a special method used to initialize objects.

It has the same name as the class and no return type.

16. What is the default constructor?

A default constructor is a no-argument constructor added by Java if none is defined.

It initializes default values.

17. Can we overload constructors?

Yes, constructors can be overloaded with different parameter lists.

This is useful for initializing objects in various ways.

18. What is the static keyword?

static indicates that a field or method belongs to the class, not instances.

It's shared among all objects.

19. What is the final keyword?

final makes a variable constant, a method non-overridable, or a class non-inheritable.

It ensures immutability or restriction.

20. What is the this keyword?

this refers to the current object instance.

It resolves naming conflicts and calls other constructors.

21. What is the super keyword?

super refers to the parent class.

It is used to call superclass constructors or methods.

22. What is an interface?

An interface defines abstract methods that a class must implement.

It supports multiple inheritance.

23. What is an abstract class?

An abstract class can have abstract and non-abstract methods.

It cannot be instantiated directly.

24. Can a class implement multiple interfaces?

Yes, Java supports multiple interface implementation.

It enables multiple inheritance of type.

25. Difference between interface and abstract class?

Interfaces can't have constructors and all methods are abstract (Java 7).

Abstract classes can have constructors and concrete methods.

26. What is a package in Java?

A package is a namespace for organizing classes and interfaces.

It avoids class name conflicts.

27. What is exception handling?

Exception handling manages runtime errors to prevent crashes.

Use try, catch, finally, and throw blocks.

28. What is the difference between checked and unchecked exceptions?

Checked exceptions are checked at compile-time (e.g., IOException).

Unchecked exceptions are not checked (e.g., NullPointerException).

29. What is the purpose of finally block?

The finally block runs regardless of exception occurrence.

It's used for resource cleanup.

30. What is a try-with-resources statement?

It automatically closes resources declared in the try() block.

It's used with AutoCloseable resources.

31. What is a String in Java?

A String is an immutable sequence of characters.

It is stored in a string pool.

32. Difference between StringBuilder and StringBuffer?

Both are mutable, but StringBuffer is thread-safe.

StringBuilder is faster in single-threaded contexts.

33. What is garbage collection?

Garbage collection automatically frees memory by deleting unreachable objects.

It is handled by the JVM.

34. What is multithreading?

Multithreading allows multiple threads to run concurrently.

It improves performance and responsiveness.

35. What is a thread?

A thread is the smallest unit of execution.

In Java, threads can be created by extending Thread or implementing Runnable.

36. What is synchronization?

Synchronization prevents multiple threads from accessing critical code simultaneously.

It ensures data consistency.

37. What is a deadlock?

Deadlock occurs when two threads wait indefinitely for each other to release resources.

Proper synchronization design avoids it.

38. What is a collection in Java?

A collection is a framework for storing and manipulating groups of objects.

It includes lists, sets, maps, etc.

39. Difference between ArrayList and LinkedList?

ArrayList is faster for random access; LinkedList is faster for insert/delete.

Both implement List.

40. What is a Map?

A Map stores key-value pairs.

Common implementations include HashMap and TreeMap.

41. What is the difference between HashMap and Hashtable?

HashMap is non-synchronized and allows null keys/values.

Hashtable is synchronized and doesn't allow nulls.

42. What is an Iterator?

An Iterator is used to traverse collections.

It allows safe removal during iteration.

43. What is generics in Java?

Generics allow type-safe data structures.

They reduce runtime errors and enable code reusability.

44. What is the use of instanceof operator?

instanceof checks if an object is an instance of a class or interface.

It helps in type checking.

45. What is method reference in Java 8?

It's a shorthand for calling a method using :: syntax.

It improves code readability in lambda expressions.

46. What is a lambda expression?

A lambda expression is a concise way to represent a functional interface.

It helps write cleaner, functional-style code.

47. What is functional interface?

A functional interface has exactly one abstract method.

It can be implemented using lambda expressions.

48. What is Stream API in Java?

Stream API processes data in a functional style.

It supports operations like filter, map, and reduce.

49. What is the difference between throw and throws?

throw is used to explicitly throw an exception.

throws declares that a method may throw exceptions.

50. What is the transient keyword?

transient prevents serialization of a variable.

It is used to skip sensitive or temporary data.
