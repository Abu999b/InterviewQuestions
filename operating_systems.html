Operating Systems Study Guide

1. What is an operating system?
An operating system (OS) is software that manages hardware resources and provides services for computer programs. It acts as an intermediary between users and computer hardware.
2. What are the types of operating systems?
  • Batch OS: Executes jobs in batches with no interaction with users.
  • Multitasking OS: Supports multiple tasks simultaneously (e.g., Windows, Linux).
  • Real-time OS: Provides immediate processing for real-time applications (e.g., embedded systems).
  • Distributed OS: Manages multiple independent computers as if they were a single system (e.g., Google's distributed systems).
  • Network OS: Manages network resources (e.g., Novell NetWare).
3. What is the difference between a process and a thread?
A process is an independent program in execution, with its own memory space. A thread is a smaller unit of execution within a process, sharing the same memory but having its own execution path.
4. What is process scheduling?
Process scheduling is the method by which the OS allocates CPU time to various processes. It involves determining which process runs at any given time.
5. What is the difference between preemptive and non-preemptive scheduling?
Preemptive scheduling allows a process to be interrupted and replaced by another, while non-preemptive scheduling ensures that a process runs to completion before another process starts.
6. What are the main process scheduling algorithms?
  • First-Come, First-Served (FCFS): Processes are executed in the order they arrive.
  • Shortest Job Next (SJN): The process with the shortest burst time is executed first.
  • Round Robin (RR): Processes are executed in fixed time slices.
  • Priority Scheduling: Processes are executed based on priority.
  • Multilevel Queue: Processes are grouped into queues based on priority or type.
7. What is a deadlock?
A deadlock is a situation where two or more processes are blocked forever, waiting for each other to release resources.
8. What are the conditions for a deadlock to occur?
The four necessary conditions are:
  • Mutual Exclusion
  • Hold and Wait
  • No Preemption
  • Circular Wait
9. What is resource allocation in OS?
Resource allocation involves managing the allocation and deallocation of system resources like CPU, memory, and I/O devices to processes and threads.
10. What is memory management?
Memory management is the process of managing the system's memory, which includes allocating memory to processes, tracking memory usage, and freeing memory when it is no longer needed.
11. What is paging in memory management?
Paging divides memory into fixed-size pages, which are then mapped to frames in physical memory. This allows processes to be loaded non-contiguously.
12. What is segmentation in memory management?
Segmentation divides memory into segments of variable sizes based on logical divisions (e.g., code, data, stack), making it easier to manage and allocate memory.
13. What is virtual memory?
Virtual memory allows the execution of processes that may not be completely loaded into physical memory. It uses disk space as an extension of RAM, swapping data between them.
14. What is thrashing?
Thrashing occurs when the OS spends more time swapping data between physical memory and disk than executing processes, resulting in severe performance degradation.
15. What is a file system?
A file system organizes and manages files on a storage device, defining how data is stored, retrieved, and manipulated.
16. What are the different types of file systems?
  • FAT (File Allocation Table): Simple, older system used in small devices.
  • NTFS (New Technology File System): Used in Windows with support for security, large file sizes, and file compression.
  • ext4: A widely used file system in Linux with features like journaling and large file support.
  • HFS+: Used by macOS for efficient file storage and management.
17. What is the difference between a file and a directory?
A file is a collection of data stored on a disk, while a directory is a container used to organize and group files and other directories.
18. What is file access control?
File access control involves setting permissions on files and directories to determine which users or processes can read, write, or execute the files.
19. What are system calls?
System calls are the interface between user programs and the OS, allowing a program to request services like process management, memory allocation, and I/O operations.
20. What is the kernel?
The kernel is the core part of the operating system that manages system resources and facilitates communication between hardware and software components.
21. What is the difference between a monolithic kernel and a microkernel?
A monolithic kernel has all core services (e.g., file management, device drivers) within the kernel space, while a microkernel provides only basic services and delegates other functions to user space.
22. What is a shell in operating systems?
A shell is a command-line interface that allows users to interact with the operating system by entering commands, scripts, or running programs.
23. What is a context switch?
A context switch occurs when the CPU switches from executing one process or thread to another, saving the state of the current process and loading the state of the new one.
24. What is an interrupt?
An interrupt is a signal to the processor, indicating an event that requires immediate attention, causing the OS to temporarily stop executing the current process and handle the interrupt.
25. What are the types of interrupts?
  • Hardware Interrupts: Generated by external devices (e.g., keyboard, mouse).
  • Software Interrupts: Generated by software instructions (e.g., system calls).
  • Timer Interrupts: Generated by the timer to allow multitasking.
26. What is a semaphore in operating systems?
A semaphore is a synchronization primitive used to control access to shared resources by multiple processes in a concurrent system. It can be binary (0 or 1) or counting.
27. What is a mutex?
A mutex (mutual exclusion) is a lock used to prevent multiple threads from simultaneously accessing shared resources, ensuring that only one thread can access the resource at a time.
28. What is a critical section?
A critical section is a part of a program that accesses shared resources (e.g., variables, data structures) and must not be executed by more than one thread at a time.
29. What is process synchronization?
Process synchronization is the coordination of processes to ensure that shared resources are accessed in a controlled manner, preventing conflicts and inconsistencies.
30. What are race conditions?
Race conditions occur when two or more processes or threads attempt to change shared data at the same time, leading to unpredictable or incorrect results.
31. What is the difference between blocking and non-blocking synchronization?
Blocking synchronization causes a thread to wait until a resource is available, while non-blocking synchronization allows the thread to continue execution even if the resource is not available.
32. What is deadlock avoidance?
Deadlock avoidance is a set of techniques used to ensure that deadlock conditions are prevented by analyzing and managing the system's resource allocation.
33. What is deadlock detection?
Deadlock detection involves periodically checking for deadlock conditions in the system and taking corrective actions, such as terminating a process or preempting resources.
34. What is the Banker's algorithm?
The Banker's algorithm is used in deadlock avoidance to allocate resources to processes safely by checking whether a resource allocation will leave the system in a safe state.
35. What is the difference between preemptive and non-preemptive multitasking?
Preemptive multitasking allows the OS to interrupt a running process and assign CPU time to another process. Non-preemptive multitasking allows a process to run until it voluntarily yields control or finishes.
36. What is a thread pool?
A thread pool is a collection of pre-created threads that are reused to perform tasks, reducing the overhead of creating and destroying threads for each task.
37. What is cooperative multitasking?
Cooperative multitasking is a type of multitasking where processes voluntarily yield control of the CPU. If a process fails to do so, it can monopolize the CPU, potentially causing system instability.
38. What is a page fault?
A page fault occurs when a process attempts to access data that is not currently in the main memory, requiring the OS to load the data from disk into memory.
39. What is the difference between internal and external fragmentation?
Internal fragmentation occurs when fixed-size memory blocks are allocated but not fully used, wasting space within the block. External fragmentation happens when free memory is scattered in small blocks, preventing large allocations.
40. What is a file descriptor?
A file descriptor is a unique identifier used by the OS to manage open files, enabling processes to read, write, or manipulate files.
41. What is the difference between hard link and soft link (symbolic link)?
A hard link points directly to a file's data on disk, while a soft link (or symbolic link) is a pointer to the file's name or path, allowing easier linking across directories and file systems.
42. What is swapping in memory management?
Swapping is the process of moving a process or part of a process from main memory to secondary storage (e.g., disk) to free up memory, and swapping it back when needed.
43. What is the purpose of a buffer in an OS?
A buffer temporarily holds data while it is being transferred between two locations, such as between the CPU and I/O devices, to prevent data loss and smooth data flow.
44. What is inter-process communication (IPC)?
IPC refers to mechanisms that allow processes to communicate and synchronize their actions, such as message passing, shared memory, or pipes.
45. What is a pipe in operating systems?
A pipe is a mechanism used for communication between processes, allowing one process's output to be sent as input to another process, often used in Unix-like systems.
46. What is a shell script?
A shell script is a series of commands written in a scripting language (such as Bash) that can be executed by a shell, automating tasks like file manipulation and process management.
47. What is the difference between a physical address and a logical address?
A physical address is the actual location in memory, while a logical address is a virtual address generated by a program that is mapped to a physical address by the memory management unit (MMU).
48. What is an I/O device in an operating system?
An I/O device (Input/Output device) is a hardware component that allows data exchange between the computer and the outside world, such as a keyboard, mouse, printer, or disk.
49. What is the role of a device driver?
A device driver is software that allows the OS to communicate with hardware devices, translating OS-level commands into specific instructions understood by the hardware.
50. What is a time-sharing system?
A time-sharing system allows multiple users or processes to share the system's resources by providing each user/process with a slice of time on the CPU, ensuring efficient resource utilization.
