React – 25 Interview Questions

1. What is React?

React is a JavaScript library for building user interfaces, developed by Facebook. It uses a component-based architecture and a virtual DOM.

2. What is JSX?

JSX stands for JavaScript XML. It allows writing HTML-like syntax in JavaScript, which React transforms into React.createElement() calls.

3. What is the virtual DOM?

The virtual DOM is an in-memory representation of the real DOM. React updates it first, then syncs changes with the actual DOM for better performance.

4. What are components in React?

Components are reusable building blocks of a React UI. They can be class-based or functional.

5. What is the difference between functional and class components?

Class components use ES6 classes and have lifecycle methods. Functional components are simpler and can use Hooks for state and side effects.

6. What are props in React?

Props (short for properties) are inputs passed to components. They are read-only and used to customize components.

7. What is state in React?

State represents data that can change over time in a component. It’s managed internally and triggers re-rendering when updated.

8. How do you update state in React?

Use the setState() method in class components or useState hook in functional ones. Direct modification of state is not allowed.

9. What are React Hooks?

Hooks let you use state and lifecycle features in functional components. Common hooks are useState, useEffect, useRef, etc.

10. What is useEffect in React?

useEffect performs side effects in functional components. It replaces lifecycle methods like componentDidMount.

11. What is the difference between useEffect and useLayoutEffect?

useEffect runs after render; useLayoutEffect runs before painting. useLayoutEffect is useful for DOM measurements.

12. What is conditional rendering?

Conditional rendering shows UI elements based on a condition. It uses JavaScript conditions like if, &&, and ternary (? :).

13. What is lifting state up in React?

Lifting state up moves state to a common ancestor component. It helps share data between sibling components.

14. What are controlled components?

In controlled components, form elements are controlled by React state. They rely on onChange handlers and state variables.

15. What is React Router?

React Router is a library for client-side routing in React apps. It allows navigation without page reloads using components like <Route> and <Link>.

16. What is the key prop in React lists?

The key prop uniquely identifies elements in a list. It helps React optimize rendering by tracking changes.

17. What is the use of useRef?

useRef creates a reference to a DOM element or a mutable value. It persists across renders without causing re-renders.

18. What are fragments in React?

Fragments let you group multiple elements without adding extra nodes to the DOM. Syntax: <></> or <React.Fragment>.

19. What is context in React?

Context provides a way to pass data through the component tree without props. It uses React.createContext() and useContext().

20. What is memoization in React?

Memoization avoids re-rendering by caching component outputs. Use React.memo for components and useMemo for functions.

21. What is React.lazy()?

React.lazy() enables code-splitting by loading components lazily. It works with Suspense to show fallback content.

22. What is the purpose of ErrorBoundary?

ErrorBoundary catches JavaScript errors in a component tree and displays a fallback UI instead of crashing the whole app.

23. What is reconciliation in React?

Reconciliation is the process of comparing old and new virtual DOM trees. React uses it to update only the changed parts of the real DOM.

24. What is the purpose of useCallback hook?

useCallback memoizes functions to prevent unnecessary re-creations. Useful in performance optimization with prop-heavy components.

25. What are higher-order components (HOC)?

An HOC is a function that takes a component and returns a new component. They’re used to reuse component logic.
