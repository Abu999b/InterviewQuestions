1What is a database management system (DBMS)?

A DBMS is a software system that manages databases by providing an interface for users to create, read, update, and delete data, ensuring data integrity, security, and efficient storage.

2What are the different types of DBMS?

Hierarchical DBMS: Data is organized in a tree-like structure (e.g., IBM's Information Management System).

Network DBMS: Data is organized in a graph structure where records are nodes, and relationships are edges (e.g., Integrated Data Store).

Relational DBMS (RDBMS): Data is organized into tables (relations) with rows and columns, and relationships are established through keys (e.g., MySQL, PostgreSQL).

Object-oriented DBMS: Data is stored as objects, similar to object-oriented programming (e.g., ObjectDB).

3What is a table in a relational database?

A table is a collection of rows (records) and columns (attributes) in a relational database. Each table represents a different entity or object in the system.

4What is a primary key?

A primary key is a unique identifier for each record in a table. It ensures that no two rows have the same value for that key.

5What is a foreign key?

A foreign key is an attribute in a table that links to the primary key of another table, establishing a relationship between the two tables.

6What are the different types of relationships in a relational database?

One-to-one: A single record in one table corresponds to a single record in another table.

One-to-many: A single record in one table corresponds to multiple records in another table.

Many-to-many: Multiple records in one table correspond to multiple records in another table (usually implemented using a junction table).

7What is normalization in DBMS?

Normalization is the process of organizing data in a database to reduce redundancy and improve data integrity. It involves dividing large tables into smaller, related tables.

8What is denormalization?

Denormalization is the process of intentionally introducing redundancy into a database by combining tables, usually to improve read performance at the cost of additional storage and complexity in updates.

9What are the normal forms in DBMS?

1st Normal Form (1NF): Each column contains atomic values, and each row is unique.

2nd Normal Form (2NF): The table is in 1NF, and all non-key attributes are fully dependent on the primary key.

3rd Normal Form (3NF): The table is in 2NF, and there are no transitive dependencies (non-key attributes do not depend on other non-key attributes).

Boyce-Codd Normal Form (BCNF): Every determinant is a candidate key.

10What is an index in DBMS?

An index is a data structure that improves the speed of data retrieval operations on a database table, allowing faster searches, inserts, updates, and deletions.

11What is a view in DBMS?

A view is a virtual table that presents data from one or more tables in a specific format. It doesn't store data itself but provides a way to query data in a user-friendly manner.

12What is a transaction in DBMS?

A transaction is a sequence of operations (e.g., read, write) that are executed as a single unit of work. It ensures consistency in the database, even in the case of failures.

13What are ACID properties?

ACID stands for:

Atomicity: Ensures that a transaction is fully completed or not executed at all.

Consistency: Guarantees that a transaction brings the database from one valid state to another.

Isolation: Ensures that transactions do not interfere with each other.

Durability: Guarantees that once a transaction is committed, it will persist even if the system crashes.

14What is a stored procedure?

A stored procedure is a precompiled collection of SQL statements that can be executed on demand. It allows for code reuse and enhances performance by reducing the need for repetitive queries.

15What is a trigger in DBMS?

A trigger is a set of SQL statements that automatically execute or "fire" when certain events (such as INSERT, UPDATE, or DELETE) occur on a specified table.

16What is a schema in DBMS?

A schema is the structure or blueprint of a database, describing how the database is organized, including tables, views, indexes, and relationships.

17What is data integrity?

Data integrity ensures the accuracy, consistency, and reliability of data in the database. It is maintained through constraints, such as primary keys, foreign keys, and unique constraints.

18What is referential integrity?

Referential integrity ensures that relationships between tables remain consistent. Specifically, it ensures that foreign key values in a child table must match primary key values in a parent table.

19What is SQL?

SQL (Structured Query Language) is a domain-specific language used for managing and manipulating relational databases, including querying, inserting, updating, and deleting data.

20What are the types of SQL commands?

DDL (Data Definition Language): Defines and manages database structure (e.g., CREATE, ALTER, DROP).

DML (Data Manipulation Language): Manipulates data in the database (e.g., SELECT, INSERT, UPDATE, DELETE).

DCL (Data Control Language): Manages permissions and access control (e.g., GRANT, REVOKE).

TCL (Transaction Control Language): Manages transactions (e.g., COMMIT, ROLLBACK, SAVEPOINT).

21What is a subquery?

A subquery is a query embedded within another query, often used in the WHERE clause to filter results based on the output of the inner query.

22What is a join in SQL?

A join is an SQL operation that combines data from two or more tables based on a related column, such as a foreign key. Types of joins include:

INNER JOIN: Returns rows with matching values in both tables.

LEFT JOIN: Returns all rows from the left table and matched rows from the right table.

RIGHT JOIN: Returns all rows from the right table and matched rows from the left table.

FULL OUTER JOIN: Returns all rows when there is a match in one of the tables.

23What is the difference between UNION and UNION ALL?

UNION: Combines the result of two or more SELECT statements and removes duplicate rows.

UNION ALL: Combines the result of two or more SELECT statements but includes all rows, including duplicates.

24What is a cursor in DBMS?

A cursor is a database object used to retrieve, manipulate, and navigate through a result set row by row, especially when the result set contains multiple rows.

25What is denormalization in DBMS?

Denormalization is the process of introducing redundancy into a database to improve performance, typically for read-heavy workloads. It involves merging tables or duplicating data for faster retrieval at the cost of additional storage and complexity in updates.

26What is the difference between clustered and non-clustered indexes?

Clustered Index: The data in the table is physically sorted and stored in the same order as the index. There can only be one clustered index per table.

Non-clustered Index: The data in the table is stored separately from the index. There can be multiple non-clustered indexes on a table.

27What is normalization and why is it important?

Normalization is the process of organizing data in a database to minimize redundancy and dependency. It ensures data integrity and improves query performance by reducing unnecessary data duplication.

28What are the advantages of using a relational database?

Data Integrity: Ensures accurate and consistent data.

Flexibility: Easy to modify and query.

Security: Allows control over access and updates.

Normalization: Helps eliminate redundancy and ensures efficient storage.

29What is a candidate key?

A candidate key is an attribute or set of attributes that uniquely identify a record in a table. A table can have multiple candidate keys, but one of them is selected as the primary key.

30What is the difference between DELETE and TRUNCATE in SQL?

DELETE: Removes rows from a table based on a condition, and it can be rolled back (if in a transaction).

TRUNCATE: Removes all rows from a table and is faster, but cannot be rolled back as it does not log individual row deletions.

31What is an entity-relationship (ER) diagram?

An ER diagram is a visual representation of the entities within a system and the relationships between them. It is used to design the structure of a database.

32What is an entity set and relationship set?

Entity Set: A collection of similar entities (objects or things) in the database that share the same attributes.

Relationship Set: A collection of relationships between entities in the database.

33What is the difference between an outer join and an inner join?

Inner Join: Returns only the records that have matching values in both tables.

Outer Join: Returns records even if there is no match. It can be a LEFT, RIGHT, or FULL OUTER JOIN, depending on which table's unmatched records are included.

34What are aggregate functions in SQL?

Aggregate functions perform calculations on a set of values and return a single value. Common aggregate functions include:

COUNT(): Returns the number of rows.

SUM(): Returns the sum of values.

AVG(): Returns the average of values.

MAX(): Returns the highest value.

MIN(): Returns the lowest value.

35What is a composite key?

A composite key is a primary key that consists of two or more columns in a table, combined to uniquely identify a record.

36What is a materialized view?

A materialized view is a database object that stores the results of a query physically. Unlike a regular view, which is dynamically created, a materialized view stores the result for improved query performance.

37What is the purpose of a rollback in DBMS?

A rollback undoes any changes made by the current transaction, returning the database to its previous consistent state. It is used to maintain the ACID property (atomicity) during errors.

38What is the difference between a full backup and an incremental backup?

Full Backup: A complete copy of all the data in the database.

Incremental Backup: Only the data that has changed since the last backup is copied, reducing time and storage requirements.

39What is a unique constraint in SQL?

A unique constraint ensures that all values in a column (or combination of columns) are distinct across the table. It allows for NULL values but only one NULL per column if the column is uniquely constrained.

40What is a relational schema?

A relational schema defines the logical design of the database, including the tables, columns, constraints, and relationships between tables.

41What is a cross join in SQL?

A cross join produces the Cartesian product of two tables. It returns all possible combinations of rows from both tables, without any condition or match.

42What is a self-join?

A self-join is a join where a table is joined with itself. It is typically used to find relationships between rows in the same table.

43What is the difference between CHAR and VARCHAR?

CHAR: A fixed-length string type that pads shorter strings with spaces to match the specified length.

VARCHAR: A variable-length string type that only uses the space necessary for the string.

44What is an optimization technique used by DBMS for query processing?

Query Optimization: The DBMS uses various strategies to improve query performance, such as index utilization, join reordering, and selection pushdown, aiming to minimize I/O operations and CPU usage.

45What is an explain plan in SQL?

An explain plan shows the execution plan for a query, detailing how the DBMS intends to access data, join tables, and optimize performance. It is used for performance tuning.

46What is the difference between OLTP and OLAP?

OLTP (Online Transaction Processing): Refers to transaction-oriented applications (e.g., banking, e-commerce) where data is frequently updated.

OLAP (Online Analytical Processing): Refers to systems used for querying and reporting large amounts of data, typically for decision-making (e.g., data warehouses).

47What is an atomic transaction?

An atomic transaction is one where either all operations are completed successfully or none are executed. It ensures the integrity of the database by preventing partial updates.

48What is SQL injection?

SQL injection is a security vulnerability that occurs when an attacker injects malicious SQL code into a query, allowing unauthorized access or manipulation of the database.

49What is the difference between a DDL and a DML command?

DDL (Data Definition Language): Defines the structure of the database (e.g., CREATE, ALTER, DROP).

DML (Data Manipulation Language): Deals with data manipulation and modification (e.g., SELECT, INSERT, UPDATE, DELETE).

50What is the role of a database administrator (DBA)?

A DBA is responsible for managing and maintaining databases. Duties include designing schemas, ensuring data security, optimizing performance, performing backups, and recovering data after failures.
