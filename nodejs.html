Node.js – 25 Interview Questions

1. What is Node.js?

Node.js is a runtime environment that allows executing JavaScript on the server side. It uses the V8 engine and is event-driven and non-blocking.

2. What is the difference between Node.js and JavaScript?

JavaScript is a language; Node.js is a runtime to run JS outside the browser. It provides libraries and modules for backend development.

3. What is the use of npm?

npm is the Node Package Manager used to install libraries and manage dependencies. You use it with commands like npm install.

4. What is a callback function in Node.js?

A callback is a function passed as an argument to another function. It executes after the main function completes, often used in async code.

5. What are modules in Node.js?

Modules are reusable blocks of code. You can import them using require() or import.

6. What is the difference between require() and import?

require() is CommonJS and used in older Node versions. import is ES6 and must be used with "type": "module" in package.json.

7. What is the event loop in Node.js?

The event loop handles asynchronous callbacks in Node.js. It allows non-blocking I/O operations.

8. What is non-blocking I/O in Node.js?

Non-blocking I/O allows other operations to run while waiting for I/O. It improves performance and scalability.

9. What is process in Node.js?

process is a global object giving info about the current Node process. It can be used for reading environment variables and handling signals.

10. What is fs module in Node.js?

The fs module is used to work with the file system. It allows reading, writing, and deleting files.

11. What is path module in Node.js?

The path module provides utilities to work with file and directory paths. It helps handle cross-platform path issues.

12. How do you handle errors in Node.js?

Use try-catch blocks for sync code and .catch() or error-first callbacks for async code. Good error handling is crucial for stability.

13. What is middleware in Node.js?

Middleware functions process requests before reaching the endpoint. Used often with frameworks like Express.js.

14. How do you read environment variables in Node.js?

Use process.env.VARIABLE_NAME. Store variables in a .env file with the dotenv package.

15. What is the use of package.json?

package.json holds metadata about the project and manages dependencies. It’s created with npm init.

16. What is the difference between synchronous and asynchronous code?

Synchronous code runs sequentially and blocks execution. Asynchronous code uses callbacks/promises to run without blocking.

17. What are streams in Node.js?

Streams are used to read/write data in chunks. They improve performance for I/O-heavy operations.

18. What is a buffer in Node.js?

Buffer is used to handle binary data. It’s useful when working with streams or files.

19. What is the difference between spawn() and exec()?

Both run child processes, but exec() buffers output while spawn() streams it. Use spawn() for large outputs.

20. What is clustering in Node.js?

Clustering allows Node to use multiple CPU cores. It improves performance in multi-core systems.

21. How do you create a server in Node.js?

Use the http module and call http.createServer(). Then use listen(port) to start it.

22. What is the global object in Node.js?

global is like window in browsers. It provides access to built-in modules and variables.

23. What is the util module in Node.js?

util provides utility functions like promisify, inherits, etc. It helps with debugging and converting callbacks to promises.

24. What is the purpose of setImmediate()?

setImmediate() schedules a function to run after the current event loop. It runs after I/O events unlike setTimeout().

25. What is the difference between process.nextTick() and setImmediate()?

process.nextTick() runs before any I/O tasks. setImmediate() runs after I/O events in the next iteration of the event loop.
