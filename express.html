Express.js – 25 Interview Questions

1. What is Express.js?

Express.js is a minimal and flexible Node.js web application framework. It provides robust features for building web and API servers.

2. How do you install Express.js?

Use the command npm install express. It installs Express into your Node.js project.

3. What is middleware in Express.js?

Middleware functions are executed during the request-response cycle. They can modify the request, response, or terminate the request.

4. What is the purpose of app.use()?

app.use() mounts middleware at a specified path and is executed for every request that matches the path.

5. What is routing in Express.js?

Routing defines how an application responds to client requests using HTTP methods like GET, POST, PUT, DELETE.

6. How do you handle GET requests in Express?

Use app.get(path, callback). The callback handles the request and sends a response.

7. What is the use of req.params?

req.params holds route parameters in the URL, used to access dynamic values from the URL path.

8. What is req.body in Express?

req.body contains data sent in the body of a POST or PUT request. You need body-parsing middleware like express.json().

9. What is the purpose of res.send()?

res.send() sends a response to the client. It can send strings, JSON, or buffers.

10. What is the difference between res.send() and res.json()?

res.send() sends any type of response. res.json() sends a JSON response with appropriate headers.

11. How do you handle POST requests in Express?

Use app.post(path, callback). You typically extract form or JSON data using req.body.

12. What is the role of next() in Express middleware?

next() passes control to the next middleware. It helps in chaining multiple middleware functions.

13. What is static file serving in Express?

Express can serve static files like HTML, CSS, and images using express.static().

14. How do you serve static files in Express?

Use app.use(express.static('public')) to serve files from the public directory.

15. What is error handling middleware in Express?

It’s a function with four parameters: err, req, res, next. It handles errors in the app globally.

16. What is CORS and how do you enable it in Express?

CORS (Cross-Origin Resource Sharing) allows requests from different domains. Use the cors package and app.use(cors()).

17. How do you connect Express to MongoDB?

Use mongoose.connect() or the native MongoDB driver. Then use Express routes to interact with the database.

18. How do you create REST APIs with Express?

Define routes using HTTP methods and handle CRUD operations. Use middleware for validation, authentication, etc.

19. How do you handle 404 errors in Express?

Add a middleware with res.status(404).send('Not Found') at the end to handle unmatched routes.

20. How can you secure an Express app?

Use HTTPS, Helmet, validation, and authentication to prevent common attacks like XSS and CSRF.

21. What is app.listen() used for?

app.listen(port) starts the server on a specified port, making your app ready to accept requests.

22. Can you create multiple routers in Express?

Yes, use express.Router() to modularize routes and organize code for larger apps.

23. How do you handle query parameters in Express?

Use req.query to access query parameters. For example, ?id=123 is accessed with req.query.id.

24. What is Helmet in Express?

Helmet is a middleware that adds security headers to protect apps from common web vulnerabilities.

25. What is the role of express.json()?

It parses incoming JSON request bodies. It's required to access req.body in POST/PUT routes.
