JavaScript Interview Questions

1. What is JavaScript?

JavaScript is a lightweight, interpreted programming language mainly used for web development.

It enables dynamic content, interactivity, and logic in web pages.

2. Is JavaScript the same as Java?

No, JavaScript and Java are different languages.

Java is compiled and strongly typed; JavaScript is interpreted and loosely typed.

3. What are data types in JavaScript?

JavaScript has primitive types like string, number, boolean, undefined, null, and symbol.

Objects are non-primitive types.

4. What is the difference between let, const, and var?

var is function-scoped, let and const are block-scoped.

const values cannot be reassigned.

5. What is hoisting in JavaScript?

Hoisting moves variable and function declarations to the top of their scope.

Only declarations are hoisted, not initializations.

6. What are closures in JavaScript?

A closure is a function that retains access to its lexical scope even when called outside it.

It helps with data privacy and functional patterns.

7. What is the difference between == and ===?

== compares values with type coercion.

=== checks for both value and type equality.

8. What is an IIFE?

IIFE stands for Immediately Invoked Function Expression.

It runs immediately and helps create private scope.

9. What are arrow functions?

Arrow functions are a concise way to write functions using =>.

They do not bind their own this.

10. What is the this keyword?

this refers to the current execution context.

Its value depends on how the function is called.

11. What is event bubbling?

Event bubbling means an event propagates from the target element up to its ancestors.

You can stop it using event.stopPropagation().

12. What is the DOM?

DOM (Document Object Model) represents the page structure as a tree of nodes.

JavaScript uses it to manipulate HTML/CSS dynamically.

13. What is the difference between null and undefined?

undefined means a variable is declared but not assigned.

null is an assigned value indicating "no value".

14. What are template literals?

Template literals use backticks (`) for strings.

They support interpolation using ${} and multiline strings.

15. What is a callback function?

A callback is a function passed into another function as an argument.

It's often used in asynchronous operations.

16. What is async/await?

async and await simplify working with promises.

They allow asynchronous code to look like synchronous.

17. What is a promise in JavaScript?

A promise represents the result of an asynchronous operation.

It can be pending, fulfilled, or rejected.

18. What is JSON?

JSON (JavaScript Object Notation) is a text format for data exchange.

It's widely used for APIs.

19. What is the difference between map() and forEach()?

map() returns a new array; forEach() doesn't.

Use map() for transformation and forEach() for iteration.

20. What are higher-order functions?

Higher-order functions take functions as arguments or return them.

Examples include map(), filter(), and reduce().

21. What is the spread operator?

The spread operator ... expands elements of an iterable.

It's used for array/object copying and merging.

22. What is destructuring in JavaScript?

Destructuring extracts values from arrays or objects into variables.

It simplifies assignment and access.

23. What is the difference between function declaration and expression?

Declarations are hoisted; expressions are not.

Function expressions can be anonymous.

24. What is NaN?

NaN means "Not a Number".

It is the result of invalid numeric operations.

25. What is typeof operator?

typeof checks the type of a variable.

It returns values like "number", "string", or "object".

26. What is the use of bind() method?

bind() sets the value of this for a function.

It returns a new function.

27. What is the use of apply() and call()?

Both invoke functions with a specified this value.

call() takes arguments individually; apply() takes them as an array.

28. What is event delegation?

Event delegation uses a parent element to handle events for its children.

It reduces the number of event listeners.

29. What is a pure function?

A pure function always returns the same result for the same inputs.

It has no side effects.

30. What is a shallow copy vs deep copy?

A shallow copy copies references; deep copy copies nested objects.

Use structured cloning or JSON.parse(JSON.stringify(obj)) for deep copy.

31. What is localStorage?

localStorage stores data in the browser with no expiration time.

It stores key-value pairs as strings.

32. What is the difference between localStorage and sessionStorage?

localStorage persists until cleared; sessionStorage lasts per session.

Both store strings only.

33. What is the event loop in JavaScript?

The event loop handles asynchronous callbacks.

It ensures non-blocking execution of code.

34. What is a stack and heap?

The stack stores function calls; the heap stores objects.

JavaScript manages both automatically.

35. What is memory leak?

A memory leak occurs when objects are not garbage collected.

It leads to increased memory usage and performance issues.

36. What is throttling and debouncing?

Throttling limits function execution over time; debouncing delays execution until inactivity.

Both optimize performance in event-heavy apps.

37. What are modules in JavaScript?

Modules allow code to be split into reusable files.

They use import and export keywords.

38. What is a symbol?

A Symbol is a unique and immutable primitive value.

Used for object property keys to avoid name collisions.

39. What is setTimeout()?

setTimeout() executes code after a delay.

It returns a timer ID for cancellation.

40. What is setInterval()?

setInterval() runs code repeatedly after a fixed time interval.

Use clearInterval() to stop it.

41. What is a generator function?

A generator yields multiple values using function* syntax.

You use next() to get each value.

42. What is strict mode?

Strict mode enforces stricter parsing and error handling.

It's enabled with "use strict".

43. What is the new keyword?

new creates an instance of an object or constructor function.

It links the prototype chain and initializes this.

44. What are falsy values in JavaScript?

Falsy values are false, 0, "", null, undefined, and NaN.

They evaluate to false in conditions.

45. What is type coercion?

Type coercion automatically converts values to the expected type.

It's implicit and can lead to bugs.

46. What is the difference between async and defer in script tags?

async loads and executes independently; defer waits for DOM parsing.

defer maintains script order.

47. What is DOMContentLoaded?

DOMContentLoaded fires when the initial HTML is fully loaded and parsed.

It's faster than window.onload.

48. What is a WeakMap?

A WeakMap holds key-value pairs with weak references.

Keys must be objects; it's not iterable.

49. What is currying in JavaScript?

Currying transforms a function with multiple arguments into a chain of functions.

It enables partial function application.

50. What is prototype in JavaScript?

Every JavaScript object has a prototype property used for inheritance.

It allows objects to share methods.
