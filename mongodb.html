MongoDB Interview Questions

MongoDB - 25 Interview Questions

1. What is MongoDB?
  • MongoDB is a NoSQL, document-oriented database.
  • It stores data in flexible, JSON-like BSON format.
2. What is a collection in MongoDB?
  • A collection is a group of MongoDB documents.
  • It is similar to a table in relational databases.
3. What is a document in MongoDB?
  • A document is a single record in BSON format.
  • It can have nested fields and arrays.
4. How is MongoDB different from SQL databases?
  • MongoDB is schema-less and stores documents, while SQL uses structured tables.
  • It's better suited for unstructured and hierarchical data.
5. What is BSON?
  • BSON stands for Binary JSON.
  • It is a binary representation of JSON used internally by MongoDB.
6. What are the advantages of MongoDB?
  • MongoDB offers flexibility, scalability, and high availability.
  • It handles unstructured data and supports horizontal scaling.
7. What is the default port of MongoDB?
  • The default port is 27017.
  • You can change it in the configuration.
8. What is a primary key in MongoDB?
  • Every document must have a unique _id field.
  • It serves as the primary key.
9. What is a replica set?
  • A replica set is a group of MongoDB servers for redundancy.
  • One acts as primary, others as secondaries.
10. What is sharding in MongoDB?
  • Sharding distributes data across multiple machines.
  • It allows horizontal scaling for large datasets.
11. What is indexing in MongoDB?
  • Indexes support efficient query execution.
  • They work like indexes in books or databases.
12. What is the aggregation framework?
  • Aggregation processes data records and returns computed results.
  • It's used for filtering, grouping, and transformations.
13. What is the difference between find() and findOne()?
  • find() returns a cursor for all matched documents.
  • findOne() returns the first matched document only.
14. How do you insert data in MongoDB?
  • Use insertOne() or insertMany() methods.
  • They add one or multiple documents respectively.
15. How do you update a document?
  • Use updateOne(), updateMany(), or replaceOne().
  • They modify existing documents based on conditions.
16. How do you delete a document?
  • Use deleteOne() or deleteMany().
  • They remove documents from a collection.
17. What is the use of $set in MongoDB?
  • $set updates the value of a field in a document.
  • If the field doesn't exist, it adds it.
18. What is the use of $match in aggregation?
  • $match filters documents based on conditions.
  • It works like the WHERE clause in SQL.
19. What is a capped collection?
  • Capped collections have a fixed size and maintain insertion order.
  • Old documents are automatically removed when full.
20. What is the ObjectId in MongoDB?
  • ObjectId is the default unique identifier for documents.
  • It includes timestamp and machine-specific data.
21. What is Compass in MongoDB?
  • MongoDB Compass is a GUI for interacting with MongoDB databases.
  • It helps visualize schema, run queries, and manage indexes.
22. What is the $group stage in aggregation?
  • $group groups input documents by a specified key.
  • It's similar to GROUP BY in SQL.
23. What is schema validation in MongoDB?
  • MongoDB allows defining rules for document structure using JSON schema.
  • It ensures data consistency in collections.
24. What is a text index?
  • A text index supports searching strings in documents.
  • It enables full-text search with operators like $text.
25. Can MongoDB handle transactions?
  • Yes, MongoDB supports multi-document ACID transactions (from version 4.0+).
  • They ensure data integrity like SQL databases.
